# .codex/templates/

Шаблоны `.codex/config.toml` для каждого из трёх блоков команды —
для участников, которые работают в **Codex** вместо Claude Code.

## Зачем

Это **жёсткая изоляция** того же типа, что у `.claude/templates/`: Codex
физически не сможет записать вне папки своего блока. Принуждение — на
уровне OS-песочницы Codex (Seatbelt на macOS, ACL на Windows), а не
инструкции агенту.

## Что внутри

Каждый файл — профиль прав Codex (ключ `default_permissions` и секция
`[permissions.raif-<block>]`), который:

- разрешает **чтение** всей ФС (`":root" = "read"`) — нужно, чтобы Codex
  внутри песочницы видел системные утилиты вне репо (`git`, `bash` и т. д.);
  на запись `":root"` прав не даёт;
- разрешает **запись** только в папку своего блока (`<block>`) и в `.git`
  (без этого `git commit` падает на `.git/index.lock`);
- закрывает **папки двух других блоков команды** через `deny`, но открывает
  чтение только их `CONTRACT.md` — это витрина, через которую блоки
  стыкуются по API;
- даёт сеть для пуша в общую копилку.

Чужие команды живут в отдельных GitHub-репозиториях, поэтому в шаблонах
больше нет правил `deny team_b/...` — соседнего репо просто нет в
файловой системе участника.

## Когда применяется

Bootstrap копирует нужный файл в `.codex/config.toml` и помечает папку
репозитория как доверенную в `~/.codex/config.toml` (`trust_level = "trusted"`).

Вручную (если bootstrap не отработал):

```bash
cp .codex/templates/config-retail.toml .codex/config.toml
```

После замены — перезапуск Codex.

## Связь с Claude

Это близнецы файлов в `.claude/templates/`. Меняешь правила доступа блока —
правь оба, чтобы Claude и Codex давали одинаковую изоляцию.
