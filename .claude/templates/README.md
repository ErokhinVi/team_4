# .claude/templates/

Шаблоны `settings.local.json` для каждого из трёх блоков команды:
`settings-<retail|cib|backend>.json`.

Команды разнесены по разным GitHub-репозиториям, поэтому в шаблонах больше
нет упоминаний `team_a` / `team_b` / `team_c` / `team_d`. Невидимость чужих
команд достигается тем, что чужого репозитория просто нет в файловой
системе участника.

## Когда применяется

Скрипт laptop-bootstrap'а (`tools/bootstrap/raif-workshop-setup.*`) копирует
выбранный шаблон в `.claude/settings.local.json` корня репозитория. Если
бутстрап не отработал — агент может скопировать сам:

```bash
# для retail
cp .claude/templates/settings-retail.json .claude/settings.local.json
```

После замены — перезапуск Claude Code App.

## Что в шаблоне

Разрешено владельцу блока:
- читать и редактировать **только свой блок** (`<own block>/**`);
- читать `CONTRACT.md` двух соседних блоков команды;
- читать `tasks/` и `seed/`;
- запускать `docker`, `docker-compose`, `git`, `python`, `curl`, `ls`,
  `cat`, `grep`, `find`.

Запрещено:
- редактировать или читать `src/`, `pyproject.toml`, `Dockerfile` соседних
  блоков команды — стык идёт **только через `CONTRACT.md`**;
- редактировать `render.yaml`, `.github/`.

Особенность блока backend: владелец может править `seed/`, чтобы добавлять
новые поля или клиентов — это часть ядра данных.

## Чужая команда

Чужого репозитория у участника не существует, поэтому отдельных deny-правил
для других команд не нужно. Симулятор и табло лидеров находятся в
организаторском репозитории и недоступны участникам.
